# 合同智能脱敏系统 (Contract Masker)

本项目是一款针对 Word 合同文档（`.docx` 格式）的高保真智能脱敏与安全还原工具。在保护隐私资产的同时，能够完美保留文档原有的段落、表格、字体字号、粗斜体及颜色等精细排版样式，并支持“免密钥本地自动还原”与“跨设备密钥包还原”的双轨制安全还原机制。

\---

## 🌟 核心技术亮点

1. **高精度 XML 格式保护**：采用跨 Run 文本区间拆分重构算法，避免常规文本替换导致的格式损坏或丢失。
2. **字体锁定与规避兼容 Bug**：根据全局规约，对重构的文字块进行 ascii、hAnsi 和 eastAsia 的统一自适应锁定，规避中西文混排字号忽大忽小与降级宋体的问题。
3. **多模态敏感词与金额比例识别**：内置 12 大标准匹配规则（新增大写金额、数值金额高精度过滤、以及百分比识别），结合本地 Jieba 语义切词初筛中文人名，并支持网页端鼠标划词一键确认/标注。
4. **双轨制无损还原**：脱敏时自动在文档属性内写入 UUID 并在本地 SQLite 数据库中保存字典以实现“一键静默还原”；同时提供备份密钥包（`.json` 格式）导出，支持在异地设备上在线无痕还原。
5. **标准OpenAPI 接口**：向外部业务系统提供全自动无状态的脱敏与还原 API，并配备交互式 Swagger UI 说明文档。
6. **亮/暗主题无缝切换**：前端基于扁平、高质感的 **Ant Design (Ant-UI)** 风格重构，支持 Light/Dark 模式一键切换并自动持久化偏好。
7. **全解耦应用个性化定制与版本控制**：支持通过本地 [config.json](config.json) 配置文件快速自定义产品名称和版本号，命令行启动 Banner、网页 Title 及 Logo（保留渐变高亮美化）将全自动同步自适应渲染更新。

## 🚀 跨平台一键启动指南

系统内置了跨平台的环境自适应引擎 `run.py`，能自动校验本地依赖并在缺失时尝试自动补齐。

### 1\. Windows 环境下启动

* **直接运行**：双击项目根目录下的 [run.bat](run.bat) 即可一键拉起服务。
* **PowerShell 命令行下运行**（特别注意）：
PowerShell 出于安全限制默认不加载当前路径的可执行文件。请先使用 `cd` 切换到本项目根目录，然后**必须带上 `.\\\\\\\\` 前缀**执行：

&#x20;   `powershell cd d:\\\\\\\\项目\\\\\\\\脱敏 .\\\\\\\\run.bat `

### 2\. Linux / macOS 环境下启动

使用终端进入本项目目录，赋予启动脚本执行权限并运行：

```bash
# 1. 赋予执行权限（仅首次运行需要）
chmod +x run.sh

# 2. 一键启动
./run.sh
```

### 3\. 服务入口划分（安全物理隔离）

* **超级管理员主页 (三合一)**：[http://127.0.0.1:8000](http://127.0.0.1:8000) (支持脱敏、还原切换，包含敏感规则配置后台)
* **专属文档脱敏页 (面向普通用户)**：[http://127.0.0.1:8000/mask](http://127.0.0.1:8000/mask) (物理剔除了 Rules 配置与还原面板，隐藏了 API 接口文档入口，实现全功能高隔离)
* **专属文档还原页 (面向普通用户)**：[http://127.0.0.1:8000/restore](http://127.0.0.1:8000/restore) (物理剔除了 Tabs 页签与配置，并隐藏了 API 接口文档入口，实现高隔离)
* **OpenAPI 接口文档 (Swagger UI)**：[http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs) (全中文参数与模型字段说明)

## ⚠️ 初次运行与生产部署核心注意事项

1. **跨平台环境自适应与依赖补齐**：

   * 本项目内置的 `run.py` 启动器会自动检测当前运行操作系统（Windows、Linux 或 macOS）。
   * **初次启动时**，启动器会自动校验本地 Python 环境并运行 `pip install -r requirements.txt` 补齐依赖。请确保设备网络畅通。若在公司内网，建议配置国内 pip 镜像源以防超时报错。
2. **跨平台启动权限与终端命令（特别注意）**：

   * **Windows (PowerShell)**：PowerShell 默认限制当前目录下未显式声明路径的脚本运行。在命令行中运行前，请先 进入项目根目录，然后**必须带上 `.\\\\\\\\` 前缀**（例如 `.\\\\\\\\run.bat`），或直接双击根目录下的 `run.bat`。
   * **Linux / macOS**：在首次启动前，必须通过终端赋予脚本执行权限：`chmod +x run.sh`，接着执行 `./run.sh` 进行启动。
3. **网络端口、主机与产品名称自定义配置**：

   * 服务默认配置在 `127.0.0.1:8000`。若该端口已被其他服务占用，启动时会报错。
   * **解决方案**：你无需修改任何源代码！直接使用记事本打开项目根目录下的 [config.json](config.json)（首次运行会自动生成），在其中修改 `"port"` 的端口数值或 `"host"` 主机地址。同时，你还可以在其中修改 `"app\\\_name"` 的值来全局自定义属于你自己的系统产品名称。启动器在运行时会自动读取该配置并生效。
4. **Word 文档独占锁冲突**：

   * 在执行脱敏或还原操作时，**被操作的 Word 文档不能在 Microsoft Word 或 WPS 中处于打开状态**。
   * Windows 系统的 Office 软件会对打开的文件加上独占写入锁，这会导致 Python 后端在读取/重写文档 XML 时抛出 `PermissionError: \\\\\\\[Errno 13] Permission denied`。
5. **本地 SQLite 数据库与无密还原生命周期**：

   * 初次运行系统后，会在根目录下自动生成 `masking.db`（SQLite 数据库文件）。
   * 该数据库存储了文档 UUID 与敏感词的真实映射。**本地“一键无密还原”功能完全依赖此数据库。** 请勿清理或删除该文件，否则已脱敏的文档将无法在本地自动配对还原（但仍可通过下载的 `.json` 备份密钥包进行手动还原）。
6. **外部系统 OpenAPI 对接生命周期管理**：

   * 通过 OpenAPI `/api/openapi/mask` 进行全自动脱敏时，接口返回的 JSON 响应中会包含 `mappings` 敏感词映射信息。
   * **调用方系统（如 OA、ERP）接收到映射后必须自行持久化存盘。** 它是后期跨机器/跨环境还原该合同唯一的钥匙。
7. **云端 Serverless / 容器部署隐患**：

   * 如果你希望将项目发布到 **Google Cloud Platform (GCP)** 的 Cloud Run 服务或 K8s 容器集群上，请务必仔细阅读项目目录 [readme/deploy\_guide.md](readme/deploy_guide.md) 中的架构修正指南，防范由于 Serverless 容器易失性（Ephemerality）导致的 SQLite 丢失与多节点负载均衡下临时上传文件丢失的问题。
